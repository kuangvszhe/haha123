# 自动上下文压缩 — Auto Compact

## 来源
源自 Claude Code 的 compact 系统。监控上下文窗口使用率，在达到阈值时自动压缩。

## 用法
后台自动运行。无需手动调用。当上下文接近限制时自动触发压缩。

## 压缩策略

### 1. 微型压缩（Micro Compact）
每次 API 调用前检查：
```
如果 已用上下文 > 上下文窗口的 80%
  → 清理不再需要的 tool result 内容
  → 保留最近的 N 条 tool result，删除旧的
  → 合并连续的系统消息
```

### 2. 会话记忆压缩（Session Memory Compact）
当上下文使用率 > 90%：
```
→ 将早期对话总结成摘要（保留关键决策和信息）
→ 用摘要替换原始对话
→ 标记被压缩的消息，避免重复压缩
```

### 3. 时间触发压缩（Time-Based MC）
当距离最后一次 AI 回复超过 1 小时：
```
→ 服务器端缓存几乎一定过期了
→ 清理旧的 tool result 以减少下次请求的 payload
→ 不影响缓存命中率（缓存已过期）
```

## 触发阈值
| 级别 | 阈值 | 行为 |
|---|---|---|
| 🟢 正常 | < 60% | 不操作 |
| 🟡 注意 | 60-80% | 微压缩：清理旧tool result |
| 🟠 告警 | 80-90% | 主动压缩+更新缓存 |
| 🔴 紧急 | > 90% | 强制压缩+截断早期消息 |

## 约束
- 给输出保留至少 20K token 空间
- 同一次会话已压缩过的部分不再压缩
- 压缩失败连续 3 次 → 停止重试（死循环保护）
