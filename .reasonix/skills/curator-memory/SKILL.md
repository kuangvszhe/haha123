# 自动记忆整理 — Curator

## 来源
源自 Hermes Agent 的 Curator 系统。自动管理记忆的生命周期：创建→活跃→过期→归档→清理。

## 用法
自动运行。无需手动调用。AI agent 在适当时机自动检查和整理记忆。

## 生命周期

```
创建 → 活跃期(30天) → 过期期(30-90天) → 归档(90天+) → 清理
```

### 各阶段规则
| 阶段 | 规则 | 自动操作 |
|---|---|---|
| **活跃**（30天内） | 至少被引用1次 | 保留 |
| **过期**（30-90天） | 未被引用 | 标记为过期，但不删除 |
| **归档**（90天+） | 未被引用 | 自动 `forget()` 删除 |
| **超载**（50条以上） | 记忆数>50 | 压缩合并同类项 |

## 触发时机
- 每完成一个大型任务后自动检查
- 记忆数超过50条时触发压缩
- 新会话开始时自动加载

## 约束
- 用户明确要求保留的记忆不受此规则影响
- type=custom 的记忆不自动删除
- 删除前会在 session 中提示"正在清理过期记忆"
