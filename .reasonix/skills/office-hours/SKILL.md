# 产品思维追问

## 来源
源自 Garry Tan (gstack) 的 `/office-hours` — YC 合伙人级的产品思维训练。

## 用法
当构思新功能、新产品、或需要对需求做深入思考时使用。模拟 YC 合伙人的追问。

## 六步追问法

### 1. 痛点验证
- "你具体遇到了什么 pain point？给我一个真实的例子。"
- "这个问题有多痛？用户愿意为此付费/花时间吗？"
- "当前用户用什么变通方案？"

### 2. 用户画像
- "谁是最核心的目标用户？"
- "他们现在用什么方法解决这个问题？"
- "你的解决方案好多少？2倍？10倍？"

### 3. 方案质疑
- "你描述的方案是目的还是手段？"
- "如果这是目的，有没有更窄的切入点？"
- "如果这是手段，真正的目的是什么？"

### 4. 范围压制
- "最小可行的版本是什么？明天能不能上线？"
- "在 80/20 法则里，哪个功能是那 20%？"
- "哪些功能可以砍掉还不影响核心价值？"

### 5. 实施评估
- "这个需要多少人/时间？你有这些资源吗？"
- "最可能失败的点在哪里？"
- "有没有不需要写代码的替代方案？"

### 6. 差异化
- "已有 N 个竞品，你的第 N+1 凭什么存在？"
- "如果你的产品成功了，为什么大公司不抄？"
- "你的护城河是什么？"

## 输出格式
```
## 产品定义
- 核心痛点：...
- 目标用户：...
- 最小方案：...

## 质疑记录
- 已反驳的假设：...
- 已接受的假设：...
- 待验证的假设：...

## 推荐实施路径
- 立即做（明天能上线）：...
- 短期（1-2周）：...
- 长期（1月+）：...
```
