# 语义记忆搜索 — Semantic Memory Recall

## 来源
源自 Claude Code 的 memdir 系统。用 AI 模型语义搜索相关记忆，而非关键词匹配。

## 用法
当需要回忆之前的对话、用户偏好、项目决策时自动触发。替代简单的 keywords 搜索。

## 与传统搜索的区别
```
传统: grep("token", .reasonix/memory/) → 只匹配包含"token"的记忆
语义: 根据当前问题的含义，找到最相关的记忆（即使关键词不同）
      例：问"API密钥怎么配" → 找到关于".env"的记忆
```

## 搜索流程
```
1. 收集当前上下文（用户问题、当前任务、已使用的工具）
2. 扫描记忆目录，读取每个记忆的标题和摘要
3. 让 AI 判断哪些记忆与当前问题相关（最多选5条）
4. 只读取被选中的记忆全文
5. 排除已在本会话中展示过的记忆（避免重复）
```

## 触发场景
| 场景 | 行为 |
|---|---|
| 用户问了一个涉及历史的问题 | 自动搜索相关记忆 |
| 开始新任务 | 自动搜索是否有关联的旧任务 |
| 用户提到之前讨论过的概念 | 自动找到对应的记忆文件 |
| 连续3轮对话使用同一工具 | 自动搜索该工具的 gotcha/warning 记忆 |

## 约束
- 一次搜索最多选5条记忆（上下文窗口限制）
- 用过的工具（当前正在用的）不搜它们的API文档记忆，但搜它们的警告/gotcha
- 记忆过多时优先选最近访问过的
