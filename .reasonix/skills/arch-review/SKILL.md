# 架构审查计划

## 来源
源自 Garry Tan (gstack) 的 `/plan-eng-review` 和 Oh My Pi 的子代理审查。

## 用法
在动手编码前，对设计方案进行架构级别的审查。

## 审查维度

### 1. 数据流图
用 ASCII 或 Mermaid 绘制数据流：
```mermaid
flowchart LR
    A[输入] --> B[处理]
    B --> C[存储]
    C --> D[输出]
```

### 2. 状态机分析
- 所有可能的状态转换
- 非法状态转换的处理
- 并发状态的处理

### 3. 错误路径矩阵
| 错误场景 | 影响 | 恢复策略 |
|---|---|---|
| 网络超时 | 数据不一致 | 重试+幂等性 |
| 输入无效 | 流程中断 | 校验+友好报错 |
| 存储失败 | 数据丢失 | 事务+回滚 |

### 4. 测试策略
- 单元测试覆盖哪些路径？
- 集成测试覆盖哪些交互？
- 端到端测试覆盖哪些场景？

### 5. 性能考量
- 预估吞吐量
- 瓶颈分析
- 缓存策略
- 扩展方案

### 6. 安全审查
- 认证机制
- 授权边界
- 数据加密
- 输入校验

## 输出格式
```
## 架构结论
- 方案可行性：[通过/有条件通过/拒绝]
- 关键风险：...
- 建议调整：...

## 审查要点
- 数据流：...
- 状态管理：...
- 错误处理：...
- 测试策略：...
- 性能：...
- 安全：...
```
