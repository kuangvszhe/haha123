# 多代理编排

## 来源
源自 OpenAI CodeX 的多代理编排、Oh My Pi 的子代理扇出和 Reasonix 的 task/parallel_tasks 工具。

## 用法
当需要并行执行多个独立子任务，或需要按流水线编排任务时使用。

## 并行扇出模式
```
1. 识别可以并行执行的独立子任务
2. 使用 parallel_tasks 同时派发
3. 每个子任务输出结果摘要
4. 汇总分析，合并结论
```

适用于：同时搜索多个模块、同时审查多个文件、同时调研多个技术方案

## 流水线模式
```
1. 任务 A → 输出结果作为任务 B 的输入
2. 任务 B → 输出结果作为任务 C 的输入
3. ...
```

适用于：先调研后执行、先生成后审查

## 主从模式
```
1. 主任务拆分工作包
2. 每个子任务用 task 派发（可设置 run_in_background）
3. 用 wait 收集所有子任务结果
4. 主任务整合结果
```

适用于：大型重构、多模块改动

## 约束
- 子任务必须有明确的范围和输出定义
- 子任务之间不能有写冲突
- 不把顺序依赖的任务并行化
