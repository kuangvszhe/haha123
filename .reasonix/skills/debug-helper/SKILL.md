# 调试辅助

## 来源
源自 Oh My Pi 的调试器集成和 OpenClaw 的沙箱能力。

## 用法
当需要调试问题（程序崩溃、逻辑错误、性能问题）时，按照此工作流执行。

## 调试流程

### 阶段 1：复现
```
1. 确认问题现象和复现步骤
2. bash 运行复现命令，记录错误输出
3. 确认是确定性还是间歇性问题
```

### 阶段 2：定位
```
1. 从错误栈追踪到源文件:line
2. 使用 grep 搜索相关代码模式
3. 使用 lsp_definition 跳转到关键符号定义
4. 使用 lsp_hover 查看类型签名
5. 使用 lsp_references 查找所有引用点
```

### 阶段 3：根因分析
```
1. 缩小到最小可复现代码片段
2. 分析数据流：输入 → 处理 → 输出
3. 检查边缘情况：空值、边界值、并发
4. 检查外部依赖：API 返回、文件状态、环境变量
```

### 阶段 4：修复
```
1. 提出最少代码改动方案
2. 优先用 edit_file 做精确编辑
3. 添加保护性检查（防御式编程）
4. 运行 lsp_diagnostics 验证编译
```

### 阶段 5：验证
```
1. 运行原有复现命令确认修复
2. 运行相关测试确保不退化
3. 添加新测试覆盖该场景（如果测试存在）
```

## 调试技巧
- **二分法**：注释掉一半代码缩小范围
- **printf 调试**：在关键路径加临时输出定位
- **对比法**：对比工作版本和故障版本的差异
- **隔离法**：将可疑代码提取到独立环境测试

## 约束
- 修复后必须移除临时调试代码
- 不要猜测根因——用证据说话
- 如果 3 步内没找到根因，用 explore 做全量调查
